To understand how the Relay works under the hood, you can explore the documentation for each package. The project is organized into modular components:

<FileTree>
- cmd/
  - [**cli**](/reference/api/cmd/cli): Command-line tools for managing the outbox.
  - [**producer**](/reference/api/cmd/producer): Example event producer.
  - [**relay**](/reference/api/cmd/relay): The main service entry point.
- internal/
  - [**config**](/reference/api/internal/config): Environment and app configuration.
  - [**container**](/reference/api/internal/container): Dependency injection and lifecycle.
  - [**publishers**](/reference/api/internal/publishers): Kafka and NATS implementations.
  - [**relay**](/reference/api/internal/relay): Core orchestration and worker loops.
  - [**storage**](/reference/api/internal/storage): Database drivers and event claiming.
  - [**telemetry**](/reference/api/internal/telemetry): Metrics and OpenTelemetry tracing.
</FileTree>
